about

I've been programming computers since the early 80s (I was six years old). I made my first website in 1994. I have a degree in Software Engineering, and 20 years' experience in the industry.

I'm a full stack web developer: I do everything from front (HTML, CSS, JavaScript/jQuery, Bootstrap) to back (PHP, Perl, MySQL), and Linux system administration. As well as writing loads of bespoke code over the years, I've done a lot of work on WordPress, and several e-commerce packages, including OSCommerce, OpenCart, and PrestaShop. Nowadays I recommend Shopify.

I set up Uniq Code Ltd in 2011 to undertake freelance and contract work. It's just me - I'm the sole employee. I only take on a couple of jobs at a time, which means I can offer you a far more personal and tailored service than a large consultancy could. If your needs are unique, you need Uniq Code!

When I started in the industry, there wasn't much prior art on the web - we had to write almost everything from scratch. That isn't the case anymore: now there's a wealth of resources out there, from simple templates to advanced frameworks to full-blown applications, but it's of very variable quality - and it's only the starting point.

Efficient and cost-effective development requires the knowledge to be able to choose the best starting materials, and to adapt and integrate them to meet the specific requirements. There is no substitute for wide-ranging experience.

I'm honest with my clients; if I'm not the right person for your job, or I'm too busy to take it on, I'll say so. Visual design is not my forte, so for briefs involving that I work alongside a fantastic designer: Loren at New Forest Web Design.

RSVP Engine

RSVP Engine is a large and complex event management system which was primarily developed by me over the course of a couple of years, but I continue to make occasional enhancements to it as requested. Written in PHP with the CodeIgniter MVC framework. There is a comprehensive admin interface, where the promoter enters the details of their events, including creating informational webpages for the event with a choice of templates to start from. Both pages and templates can be edited within the system using a WYSIWYG editor; the thumbnails (as shown above) are automatically re-generated when their template or page has changed.

The system also handles emails for invitations, registration confirmations etc, again with multiple (editable) templates to use as starting points. The customer-facing site then displays the info for the event(s) that the logged in user has been invited to (and/or any public events), and provides a form for them to register if they are coming, and takes their catering order if applicable.

The templates allow for the system to be fully "white label", with each promoter able to use their own branding, colour scheme etc.

The catering products are fully configurable by the promoter, including complex rules that were developed for the software's first major customer, Perth Arena in Australia, who have been using the system since 2014. They required that some products could only be ordered in pre-defined quantites, while others can have any quantity entered. The total is added up as the user makes their selections. Registrations and catering orders are then collated and can be accessed by the promoter via the admin interface.

I also designed a RESTful API so that the system can be integrated with other applications. And I installed, configured and do ongoing administration of, a pair of Linux production servers, including Postfix and Dovecot for mail, Bind for DNS, and Apache for the website, plus bi-directional MySQL replication, cross-synchronisation of the web content, and remote backups.

In 2017 I made the application multilingual, and moved the website and outgoing email to AWS ElasticBeanstalk.
